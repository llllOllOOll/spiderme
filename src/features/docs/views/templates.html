Templates

Spider's template engine is embedded at compile time via @embedFile. Supports variable interpolation, conditionals, loops, named blocks, filters, and HTMX-aware rendering.

Variable interpolation

Use {{ }} to interpolate values:

html
{% raw %}
<h1>{{ title }}</h1>
<p>Welcome, {{ user_name }}!</p>
<p>Items: {{ count }}</p>
{% endraw %}
zig
const html = try spider.template.render(tmpl, .{
    .title = "Dashboard",
    .user_name = "Alice",
    .count = 42,
}, alc);

Conditionals

Use {% if %} for conditional rendering:

html
{% raw %}
<!-- Basic condition -->
{% if is_admin %}
<p>Admin panel access</p>
{% endif %}

<!-- Negation -->
{% if !is_guest %}
<p>Welcome back!</p>
{% endif %}

<!-- Equality -->
{% if status == "active" %}
<span class="badge green">Active</span>
{% endif %}

<!-- Inequality -->
{% if role != "banned" %}
<a href="/profile">View Profile</a>
{% endif %}

<!-- Logical AND -->
{% if is_logged_in and is_premium %}
<p>Premium features unlocked!</p>
{% endif %}

<!-- Logical OR -->
{% if is_admin or is_moderator %}
<button>Moderate</button>
{% endif %}

<!-- Else branch -->
{% if has_permission %}
<a href="/edit">Edit</a>
{% else %}
<p>No edit access</p>
{% endif %}
{% endraw %}

Boolean strings "true", "1", "yes" evaluate to true. "false", "0", "no" evaluate to false.

Loops

Use {% for %} to iterate over a slice:

html
{% raw %}
<ul>
{% for item in items %}
<li>{{ item.name }} - {{ item.price }}</li>
{% endfor %}
</ul>
{% endraw %}
zig
const html = try spider.template.render(tmpl, .{
    .items = items,
}, alc);

Named blocks & includes

Define blocks with {% block %} and include them with {% template %}:

html
{% raw %}
{% block "header" %}
<nav>Site Navigation</nav>
{% end %}

{% block "content" %}
<p>Page content here</p>
{% end %}
{% endraw %}
html
{% raw %}
<!-- In your layout -->
<header>{% template "header" %}</header>
<main>{% template "content" %}</main>
{% endraw %}

Filters

Apply filters with the | pipe operator:

html
{% raw %}
<p>Hello {{ name | default:"Guest" }}!</p>
<p>Value: {{ amount | default:"0.00" }}</p>
{% endraw %}
zig
// If name is empty/null, "Guest" is used instead
const html = try spider.template.render(tmpl, .{
    .name = "", // or null
}, alc);

Rendering

Spider provides three rendering functions:

render

Renders a template string directly. No layout, no blocks.

zig
const html = try spider.template.render(tmpl, data, alc);

Response.html

Wraps a raw []const u8 in an HTML response. No template rendering.

zig
return spider.Response.html(alc, content);

chuckBerry

The recommended function for handlers. Resolves the template by name, handles {% extends %}, and detects HTMX requests automatically:

zig
pub fn index(alc: std.mem.Allocator, req: *spider.Request) !spider.Response {
    const context = try buildContext(alc, req);
    return spider.chuckBerry(alc, req, "home_index", context);
}

chuckBerry automatically:

  • Normalizes template name ("home/index" → "home_index")
  • Resolves {% extends %} and merges layout
  • HTMX request (HX-Request): returns {% block "content" %} only
  • Normal request or HX-Boosted: returns {% block "base" %} with full layout
zig
// How chuckBerry handles requests internally (simplified)
const is_htmx = req.headers.get("HX-Request") != null;
const is_boosted = req.headers.get("HX-Boosted") != null;
if (is_htmx and !is_boosted) {
    // Partial update: returns {% block "content" %} only
    return renderBlock("content", data, alc);
} else {
    // Full page: returns {% block "base" %} with layout
    return renderBlock("base", data, alc);
}

Auto-generate Templates

zig
const gen = b.addRunArtifact(spider_dep.artifact("generate-templates"));
gen.addArg("src/");
gen.addArg("src/embedded_templates.zig");
