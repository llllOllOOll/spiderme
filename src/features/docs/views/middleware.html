Middleware

Middleware functions run before every request handler. Chain them with server.use(fn). Each middleware receives the allocator, request, and a next function to continue the chain.

zig
// Built-in middleware
const web = spider.web;

server.use(web.corsMiddleware)
.use(web.loggerMiddleware)
.use(web.authMiddleware)
.get("/", homeHandler)
.listen() catch |err| return err;

// Custom middleware
fn authMiddleware(
alc: std.mem.Allocator,
req: *spider.Request,
next: spider.NextFn,
) !spider.Response {
const key = req.header("x-api-key");
if (key == null) {
var res = try spider.Response.text(alc, "Unauthorized");
res.status = .unauthorized;
return res;
}
return next(alc, req);
}

Injecting Data via Middleware

Middleware can inject data into the request using req.params.put(), making it available to all downstream handlers. This is how smoney's auth middleware passes the authenticated user_id to every protected route.

// From smoney/src/auth/middleware.zig
// Middleware: inject user_id after verification
pub fn authMiddleware(
alc: std.mem.Allocator,
req: *spider.Request,
next: spider.NextFn,
) !spider.Response {
// Check public paths first
const PUBLIC_PATHS = [_][]const u8{ "/login", "/assets" };
for (PUBLIC_PATHS) |path| {
if (std.mem.eql(u8, req.path, path)) return next(alc, req);
}

// Verify JWT token
const jwt_secret = std.mem.span(std.c.getenv("JWT_SECRET") orelse "");
const token = req.cookies.get("session") orelse {
return spider.Response.redirect(alc, "/login");
};
const claims = spider.auth.jwtVerify(alc, token, jwt_secret) catch {
return spider.Response.redirect(alc, "/login");
};

// Inject user_id so all handlers can access it
const id_str = try std.fmt.allocPrint(alc, "{d}", .{claims.sub});
try req.params.put("_user_id", id_str);

return next(alc, req);
}

Read the injected data in any handler:

// Any protected handler can access injected data
fn dashboardHandler(alc: std.mem.Allocator, req: *spider.Request) !spider.Response {
const id_str = req.params.get("_user_id") orelse return error.Unauthorized;
const user_id = try std.fmt.parseInt(i32, id_str, 10);

// user_id is now available — query database, render template, etc.
const user = try UserRepository.findById(user_id);
return spider.renderView(alc, req, dashboard_view, .{ .user = user });
}
Key points:
  • req.params.put(key, value) injects a string value into the request
  • req.params.get(key) reads it back in any handler
  • Values are scoped to the current request — no global state
  • Convention: prefix injected keys with _ to distinguish from URL params (e.g. _user_id, _locale)

smoney also uses this pattern for _locale in the locale middleware to pass the user's preferred language to all handlers.
